# `v0.5.0` Release Notes

`v0.5.0` extends the public preview with a more complete compare-mode workflow, broader app polish, corrected pinwheel geometry, palette-accurate tiling thumbnails, and the first tag-triggered automated release path.

## Highlights

- Added compare mode as a practical exploration surface, with seed drawing, live previews, shareable begin/end links, open-in-place result loading, and clearer modal actions.
- Promoted `pinwheel` into the main `Aperiodic` picker group after correcting the Conway-Radin subdivision and accepting the rendered patch in manual review.
- Fixed tiling picker thumbnails so sampled geometry and colors better match the actual canvas-rendered tilings.
- Hardened CI and release automation, including pinned GitHub Actions, coverage gates, Dependabot automation, generated-file freshness checks, and a tag-triggered Release workflow.
- Strengthened Python dependency integrity by regenerating hashed lockfiles and consolidating Python tool configuration into `pyproject.toml`.

## User-Facing Changes

- Compare mode now supports drawing a seed, previewing named shapes, copying begin-state links, and opening comparison results directly in the main workspace.
- The main app chrome, mobile toolbar, drawer sections, tiling picker, rule search, and first-action hints were polished for faster scanning and repeated use.
- Tiling thumbnails now use the same palette tokens as the canvas for palette-backed families, including Rhombille, Deltoidal Hexagonal, Deltoidal Trihexagonal, Basketweave, Herringbone, Stein 14 Pentagonal, and related mixed tilings.
- Square, Hexagonal, Triangular, and Type 7 Pentagonal thumbnails now use the neutral dead-cell fill expected from the canvas renderer.
- `pinwheel` is no longer labeled experimental. `pinwheel-2-1` remains experimental pending its own manual visual review.
- The duplicate triangle-hexagon Life rule entry was consolidated so the rule picker exposes one canonical mixed-tiling rule.

## Topology Notes

- `pinwheel` now keeps every substitution descendant congruent to the canonical `1:2:sqrt(5)` triangle. The reference verifier enforces this with per-tile side-ratio checks for both `pinwheel` and `pinwheel-2-1`.
- The June 2026 pinwheel correction fixed non-canonical child vertex ordering that previously sheared depth-2-and-deeper descendants even while preserving equal areas and balanced chirality.
- The corrected `pinwheel` patch is genuinely non-edge-to-edge, so it keeps the same polygon-union surface waiver class as `pinwheel-2-1`; overlap, edge multiplicity, and graph connectivity checks remain strict.
- Palette-backed thumbnail data now follows the shared family dead-palette manifest instead of generic preview fill slots.

## Tooling And Quality Improvements

- Added `.github/workflows/release.yml`, triggered by semantic version tags. The workflow validates release prerequisites, rebuilds frontend and standalone artifacts, checks bundle size, smoke-tests the standalone bundle with Chromium, and publishes the GitHub Release with the standalone zip attached.
- Added `python -m tools repo release-check` for pre-publish and post-publish validation of release notes, release-facing docs, local/remote tags, GitHub Releases, and the latest-release pointer.
- Added `python -m tools repo generated-check` for generated docs, bootstrap fixture, frontend topology fixture, and reference fixture freshness.
- Added backend and frontend coverage gates, Dependabot auto-merge for green minor/patch updates, commit-pinned GitHub Actions, `merge_group` CI support, and a Windows backend unit job.
- Regenerated Python lockfiles with hashes and consolidated Ruff, mypy, pytest, and coverage configuration into `pyproject.toml`.

## Known Limitations

- This preview line is still a source release and GitHub Pages demo, not an npm or PyPI package.
- The standalone demo still depends on CDN-hosted Pyodide assets and is not an offline bundle.
- Some aperiodic families still ship as `canonical_patch`, exact-affine representative patches, or otherwise documented approximations rather than full symbolic or literature-canonical substitution implementations.
- `pinwheel-2-1` remains experimental until manual visual review accepts its exact-affine implementation.
