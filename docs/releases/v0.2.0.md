# `v0.2.0` Release Notes Draft

`v0.2.0` expands the public preview with a new topology, a more usable editor, stronger standalone-browser behavior, and tighter validation around the shipped demo and source checkout.

## Highlights

- Added the `kisrhombille` tiling family, the dual of truncated trihexagonal `4.6.12`, and wired it through the shipped topology catalog.
- Reworked the editor and control surface with a focused drawer layout, progressive-disclosure paint tools, and clearer keyboard-shortcut access.
- Improved standalone-browser reliability and persistence behavior.
- Strengthened validation with broader frontend typing, broader Python coverage enforcement, standalone provenance checks, and CI hardening.
- Expanded contributor documentation for adding rules, presets, and topologies, including additional safeguards learned from shipping new topology families.

## User-Facing Changes

- New topology: `kisrhombille`
- More direct paint and erase interaction
- Drawer sections reorganized around Simulation, Topology, Patterns, Display, and Advanced concerns
- Keyboard shortcuts surfaced in the UI
- Better rectangle-brush preview behavior
- Standalone persistence and reload behavior made more dependable

## Tooling And Quality Improvements

- Backend coverage moved to `pytest-cov`
- Frontend linting and formatting checks added and tightened
- Documentation link checking replaced with Linkinator
- Tiling-preview generation and topology-addition tooling expanded
- Render-review and browser-check tooling hardened across platforms, including Windows-specific path and tempdir handling
- CI coverage, bundle-budget, and standalone validation paths tightened

## Topology Verification Notes

- `penrose-p2-kite-dart`, `penrose-p3-rhombs`, and `robinson-triangles` are currently documented as known deviations rather than canonical substitution implementations in the shipped preview line.
- `dodecagonal-square-triangle` still ships as a structurally exact periodic square-triangle generator, not the canonical Schlottmann quasi-periodic marked-prototile construction.
- `pinwheel` remains `Experimental` pending stronger manual visible review.

## If Penrose Work Lands Before Release

- Replace the first Topology Verification Notes bullet with a specific statement about what changed and what still remains a known deviation.

## Known Limitations

- This preview line is still a source-release and GitHub Pages demo, not an npm or PyPI package.
- The standalone demo still depends on CDN-hosted Pyodide assets and is not an offline bundle.
- Some aperiodic families still ship with explicit verification caveats or known deviations, and those limitations should remain visible in public-facing docs.
