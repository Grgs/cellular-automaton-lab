# `v0.3.0` Release Notes

`v0.3.0` extends the `v0.2.0` preview line with one new periodic convex pentagonal tiling, a second Penrose P1 construction, and stricter release-surface guardrails around frontend build freshness and documentation accuracy.

Current catalog note: Penrose P1 is now exposed as one public `penrose-p1` family with `Distributed` and `Boat-Star` construction modes. The older construction-specific ids remain only as implementation geometry keys and legacy import aliases.

## Highlights

- Added `type-7-pentagonal` as a new periodic convex pentagonal family in the existing `periodic_face` architecture.
- Added a centered singular-pentagrid Penrose P1 construction that emits the full `pentagon` / `diamond` / `boat` / `star` vocabulary.
- Kept the older distributed non-singular P1 manifestation and clarified the distinction between the two construction choices.
- Guarded local Flask startup against stale `static/dist` frontend bundles so source edits and served assets do not silently diverge.
- Refreshed release-facing documentation, source citations, verification tables, changelog coverage, and bootstrap metadata to match the shipped tiling catalog.

## User-Facing Changes

- New topology: `type-7-pentagonal`
- New Penrose P1 construction mode: `Boat-Star`
- Clarified Penrose P1 construction mode: `Distributed`
- The convex pentagonal catalog now spans Cairo, Prismatic, Floret, Type 7, Snub Square Dual, and Penrose P1
- Local app startup now fails fast with a clear instruction when the checked-in/server bundle is older than the authored frontend source

## Topology Notes

- `type-7-pentagonal` is a periodic convex pentagonal family with a compact descriptor-driven repeat unit and stable `periodic_face` ids, so it fits the existing validation and preview pipeline cleanly.
- The Penrose P1 `Boat-Star` construction is a `canonical_patch` implementation geometry built from the singular all-zero de Bruijn pentagrid crop. It is deterministic, gap-free, and validator-clean, but it is still a canonical multigrid manifestation rather than the literal decorated six-state substitution implementation.
- The Penrose P1 `Distributed` construction remains the non-singular `P3 -> P1` vertex-merge manifestation. It distributes pentagon and star regions across the patch and does not emit canonical P1 boats.
- `pinwheel` remains `Experimental`.
- `dodecagonal-square-triangle` still ships as a decorated periodic `3.12.12` generator rather than the canonical Schlottmann quasi-periodic square-triangle tiling.

## Tooling And Quality Improvements

- Added a startup-time freshness guard for the server-served frontend build
- Refreshed bootstrap fixtures and release-facing metadata after adding the second Penrose P1 construction
- Extended periodic and aperiodic documentation so source references, verification tables, changelog coverage, and known-limit notes stay aligned with the shipped families

## Known Limitations

- This preview line is still a source release and GitHub Pages demo, not an npm or PyPI package.
- The standalone demo still depends on CDN-hosted Pyodide assets and is not an offline bundle.
- Some aperiodic families still ship as `canonical_patch` or otherwise documented approximations rather than full symbolic or literature-canonical substitution implementations.
